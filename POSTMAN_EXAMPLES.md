## 1. Расчет доставки с подъемом на этаж (склад → дверь, экономичная)

**Endpoint:** `POST http://localhost:8080/delivery/calculate`

**Headers:**
```
Content-Type: application/json
```

**Body (JSON):**
```json
{
  "length_cm": 40,
  "width_cm": 30,
  "height_cm": 20,
  "weight_kg": 3.5,
  "to": "дверь",
  "from_address": "Москва, ул. Тверская, д. 10",
  "to_address": "Санкт-Петербург, Невский пр., д. 28",
  "shipment_date": "2025-12-02",
  "speed": "economy",
  "extra_services": {
    "insurance_value": 10000,
    "need_packing": false,
    "need_courier": false,
    "need_documents": false,
    "need_storage": false,
    "need_unloading": true,
    "floor": 5,
    "has_elevator": false
  }
}
```

**Описание:**
- Доставка склад → дверь
- Экономичная доставка (speed: economy)
- Подъем на 5 этаж без лифта
- Страхование на 10000 рублей

**Ожидаемый результат:**
- Тарифы СДЭК: только 62, 121, 122, 123, 748, 749, 750, 751
- В запросе к СДЭК должна быть услуга `GET_UP_FLOOR_BY_HAND` с параметром `"5"`
- Варианты отсортированы по цене
- Тарифы с "постомат" исключены

**Описание:**

## Как проверить работу

### 1. Проверка подъема на этаж
После отправки запроса проверьте логи или отладочный вывод запроса к СДЭК. В поле `services` должен быть элемент:
```json
{
  "code": "GET_UP_FLOOR_BY_HAND",
  "parameter": "5"  // или "1" если этаж не указан
}
```

### 2. Проверка фильтрации по speed
- При `speed: "economy"` должны быть только тарифы: 62, 121, 122, 123, 748, 749, 750, 751
- При `speed: "express"` должны быть только тарифы: 480, 481, 482, 483
- Без `speed` или с другим значением - все тарифы СДЭК

### 3. Проверка фильтрации по to
- При `to: "дверь"` - только тарифы "склад-дверь"
- При `to: "склад"` - только тарифы "склад-склад"

### 4. Проверка сортировки
Все варианты в ответе должны быть отсортированы по цене (от меньшей к большей), независимо от провайдера.

### 5. Проверка исключения постоматов
В ответе не должно быть тарифов с "постомат" в названии.

## Примечания

- Все даты в формате `YYYY-MM-DD`
- Цены возвращаются в копейках (int64)
- Даты доставки в формате ISO 8601
- Если `floor` не указан при `need_unloading: true`, используется 1 этаж
- Поле `has_elevator` пока не используется в запросах к СДЭК, но сохранено в модели для будущего использования
